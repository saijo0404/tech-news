# Micro-Agent: Beat Frontier Models with Collaboration inside Model API

- **來源**: vLLM Blog
- **發布日期**: 2026-06-29
- **原文連結**: https://vllm.ai/blog/2026-06-29-micro-agent-frontier-models

## 核心主題
vLLM 提出 **Semantic Router** 概念，將微代理（micro-agent）協作機制嵌入模型服務層——開發者只需呼叫一個 `vllm-sr/auto` 模型名稱，路由器即可在幕後自動選擇合適的協作模式（如信心升級、平行評級、ReMoM、Fusion、Workflows），以一個 API 表面擊敗單一流行模型。

## 關鍵重點
- **路由器從「模型選路」進化到「能力建構」**：路由器不再只是決定哪請求送哪模型，而是將單一模型 API 呼叫轉化為有界協作。vLLM 提供五種 Looper 協作模式：①**Confidence**（信心升級）：先試便宜模型，若信心分數不足則升級到更強模型，只在難題上花錢。②**Ratings**（評級聚合）：在硬式併發上限下平行執行多個模型，以評級權重聚合結果。③**ReMoM**（重複混合推理）：發散多個推理嘗試，等待最低成功法定人數後由綜合模型合併證據。④**Fusion**（分歧整合）：將獨立模型回應作為證據，讓裁判模型分析一致與矛盾點後產出單一答案。⑤**Workflows**（工作流程）：以計畫器、修補器、驗證器、最終化器等微代理角色執行有界步驟。
- **任務形狀決定協作模式**：不同基準測試需要不同的協作策略——GPQA-Diamond（科學多選）用 ReMoM 搭配嚴格答案格式；LiveCodeBench（程式碼）偵測約束條件與隱藏測試風險後選擇程式碼型迴圈；Humanity's Last Exam（形式推理）在 ReMoM、Fusion 或備份路徑間動態選擇。路由器以 `vllm-sr/auto` 為公開模型名稱，內部根據難度、風險、合約壓力、延遲與成本信號自動匹配模式，無需應用程式端自行編寫代理圖譜。
- **評分卡證明協作勝於單模型**：在 LiveCodeBench 上 VSR Closed 達成 **92.6%**（超越 Fugu Ultra 92.0、GPT-5.5 90.7、Opus 4.8 90.3）；GPQA-Diamond 達 **96.0%**（超越 Fugu Ultra 95.5、Gemini 3.1 Pro 94.3）；HLE 達 **50.0%**（與 Fugu Ultra 並列第一）。VSR Hybrid（混合開放與閉源模型）在 HLE 上達 47.1%，證明開放與閉源模型可在同一服務抽象下參與。

## 結論
vLLM 的核心理念是：微代理不應該只活在應用程式邏輯中，而應該是服務層的原生能力。路由器已擁有微代理所需的一切——模型別名、供應商政策、憑證、成本元數據、重試、逾時、追蹤與 OpenAI 相容回應語義——將協作機制內建於路由器，讓「一個模型 API 呼叫」變為「一支紀律嚴明的小團隊」，既保持 API 穩定，又讓系統在不改變客戶端整合的情況下持續變強。
