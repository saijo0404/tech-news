# What building Shippy taught us about building agents

- **來源**: Hugging Face Blog
- **發布日期**: 2026-07-15
- **原文連結**: https://huggingface.co/blog/allenai/shippy-tech-blog

## 核心主題
這篇文章探討了 Ai2 團隊如何從高風險的海洋 AI 代理 Shippy 的開發經驗中，學習到構建可靠 AI 代理的關鍵原則。

## 關鍵重點
- **代理架構三要素**：Shippy 由「靈魂」（系統提示定義行為邊界）、「技能」（處理特定請求的能力）和「配置」（運行時設定如 LLM 選擇器）組成，其中技能和靈魂被編譯成可版本控制的 Docker 映像。
- **確定性工具層面**：為了管理非確定性的模型行為，團隊開發了專用的 CLI 工具層，將複雜的 API 呼叫簡化為可預測的命令，並將結果寫入本地 JSON 檔案以避免管道緩衝限制。
- **沙盒隔離機制**：每個用戶擁有獨立的 Kubernetes 部署沙盒，確保用戶數據隔離，包括 JWT 驗證、文件隔離和網絡限制，支持多用戶同時運行。
- **完整評估系統**：建立了基於專家撰寫情境和評分標準的評估框架，使用 LLM 審判對代理表現進行評分，並通過 Harbor 框架在真實環境中測試。

## 結論
Shippy 的開發經驗為未來代理設計提供了重要指導，包括可審計的邊界設定、工具確定性、數據隔離和持續評估。這些原則將應用於其他環境平台，如 EarthRanger 和 OlmoEarth。團隊正進一步開發代理驅動的 UI 控制、模型路由和跨線路記憶功能。
