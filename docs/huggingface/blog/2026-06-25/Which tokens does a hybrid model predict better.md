# Which tokens does a hybrid model predict better

- **來源**: Hugging Face Blog（與 AllenAI 合作）
- **發布日期**: 2026-06-25
- **原文連結**: https://huggingface.co/blog/allenai/hybrid-token-prediction

## 核心主題
AllenAI 與 Hugging Face 透過對比 Olmo 3（純 Transformer）與 Olmo Hybrid（混合架構）在逐詞元（token）層級的預測表現，揭示混合模型在含義詞元上的優勢，以及 Transformer 在精確重複與括號匹配上的專長。

## 關鍵重點
- **混合模型在含義詞元上表現更強**：Olmo Hybrid 在名詞、動詞、形容詞等承載語義的「實詞」上預測損失較低（差距約 0.04），遠高於虛詞（如 "the"、"of"、"is"）的差距（約 0.02）。混合架構的循環層（RNN）擅長追蹤隨閱讀過程逐步變化的資訊，使其在理解句子核心含義上具備優勢。
- **Transformer 專精於精確重複與括號匹配**：混合模型的優勢在兩種情境下幾乎消失——（1）下一個詞元是前文逐字重複的內容（n-gram 重複），此處 Transformer 的注意力機制能直接回溯並精確取回早期資訊；（2）閉合括號的預測，因為括號匹配可由注意力機制單獨充分表示。純循環模型則在重複詞元上落後兩者。
- **過濾詞元損失作為架構評估新指標**：研究提出使用「過濾詞元損失」（filtered token losses）作為預訓練中比較不同架構的細粒度評估方法。在 1B 參數模型的預訓練實驗中，含義詞元（非重複）上混合模型與純循環模型皆超越 Transformer，而重複詞元上純循環模型落後。這顯示早期預訓練階段即可透過細粒度分析揭示架構差異。

## 結論
單一整體損失（overall loss）不足以區分 Transformer 與混合架構的真實優劣。透過逐詞元層級的細粒度分析，研究揭示了混合架構在語義追蹤與循環狀態維持上的獨特優勢，以及注意力機制在精確回憶上的不可替代性。這些發現將指導未來混合架構的設計方向——理解每個組件擅長什麼，並據此優化整體設計。
