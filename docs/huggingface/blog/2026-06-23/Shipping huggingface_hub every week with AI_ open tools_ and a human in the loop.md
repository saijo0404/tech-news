# Shipping huggingface_hub every week with AI, open tools, and a human in the loop

- **來源**: Hugging Face Blog
- **發布日期**: 2026-06-23
- **原文連結**: https://huggingface.co/blog/huggingface-hub-release-ci

## 核心主題
Hugging Face 團隊以開源工具與開放權重模型建構每週自動發布管道——AI 負責起草發布筆記，確定性程式碼驗證完整性，人類審核最後決定——將 huggingface_hub 的發布頻率從每 4-6 週一次提升至每週一次，同時降低每輪發布成本至約 0.25 美元。

## 關鍵重點
- **自動化與 AI 起草的分工設計**：傳統發布流程中，版本跳號、標籤建立、PyPI 發布等機械性步驟由 CI 處理，但撰寫發布筆記需數小時手動整理 PR；新流程以 GitHub Actions 驅動，使用 OpenCode 執行時與 GLM-5.2 開放權重模型自動起草發布筆記與 Slack 公告，人類只需十五分鐘編輯而非半天撰寫。模型草稿由確定性 Python 腳本驗證——提取 PR 編號清單作為真實值，比對模型輸出，若有遺漏或額外則自動要求修正，直到完全吻合。
- **Grounding 與技能驅動提示**：為防止模型憑標題瞎編，系統同步拉取 PR 修改的 `docs/*.md` 統一差異檔作為上下文，確保模型引用的程式碼範例與文件一致。提示詞以 Skills（SKILL.md 檔案）形式存在於版本控制中，如同新進人員引導文件，可替換範本、保留結構。發布過程還包含下游測試分支自動開啟、發布後註記 PR、同步 CLI 文件等自動化步驟。
- **安全、開源與可複製的架構**：發布使用 PyPI Trusted Publishing（OIDC 短期令牌替代長期 API Token）、OpenCode 執行時版本錨定並 SHA256 校驗、所有工具與模型皆開源可自託管。每輪發布成本約 0.25 美元（按量計費），並將 AI 原始草稿與人類編輯版並存儲存至 HF Bucket，累積比對資料以持續精進技能。任何維護者皆可分叉工作流程檔案、重寫技能 Markdown、設定兩個環境變數即可複製。

## 結論
這項工作的核心啟示不是「讓 AI 去做」，而是「讓模型起草、讓確定性程式碼驗證、讓人類決定」——將人類從繁瑣的筆記撰寫中解放，專注於校準、潤飾與品質把關。發布頻率從每月一次到每週一次，發布筆記品質不降反升，下游錯誤更早浮現，貢獻者迴圈縮短。整個流程完全由開源工具與開放模型建構，成本趨近於零，為所有 Python 函式庫維護者提供了一條可立即採用的自動化發布藍圖。
