# Architecting a trusted agentic platform with graph technologies: A Yahoo case study

- **來源**: Google Cloud Blog
- **發布日期**: 2026-06-15
- **原文連結**: https://cloud.google.com/blog/products/databases/graph-technologies-underpin-yahoo-system-of-action/

## 核心主題
Yahoo 與 Google Cloud 合作，利用圖形技術打造 Seller Agent 數位媒體購買平台，展示企業如何以知識圖譜與情境圖譜的雙圖架構，建構快速、透明且可審計的 AI 自主執行系統。

## 關鍵重點
- **雙圖譜架構：知識圖譜負責執行，情境圖譜負責記憶**：知識圖譜（Spanner Graph）將廣告產品、受眾、庫存、合約與治理政策以關聯式實體建模，讓 AI 智能體在單一查詢中即可評估業務現實與合規限制；情境圖譜（BigQuery Graph）則完整記錄每次決策的完整脈絡，將每項行動轉化為可查詢的證據鏈，使審計從事後翻日誌變成即時單一查詢即可解釋。
- **從數週手工作業縮短至數秒自主執行**：透過 GKE 上的多智能體系統與 ADK 協調，買方需求被分解為庫存發現、受眾配對、預測、定價、治理審查等專業任務，由智能體透過 A2A 協定協作完成，原本需要跨部門協調數週的流程，現在只需數秒即可生成可執行的活體廣告活動。
- **可審計、可治理的信任基礎設施**：Yahoo 的設計強調「信任」而非僅是「智慧」——決策依據業務現實而非機率模型，每項行動都有不可篡改的追蹤紀錄，若廣告主詢問特定方案被選中的原因，系統可透過情境圖譜即時回覆，滿足監管級治理需求。

## 結論
Yahoo 的案例證明，企業自主 AI 系統的競爭壁壘不在於使用什麼語言模型，而在於擁有什麼樣的業務圖譜與治理歷史。知識圖譜讓 AI 扎根於真實業務，情境圖譜讓每次決策透明可追溯，這套雙圖譜模式可延伸至金融交易、供應鏈等任何需要高風險決策且要求可解釋性的產業。
