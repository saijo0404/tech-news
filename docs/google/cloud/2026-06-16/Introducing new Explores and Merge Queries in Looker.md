# Introducing new Explores and Merge Queries in Looker

- **來源**: Google Cloud Blog
- **發布日期**: 2026-06-16
- **原文連結**: https://cloud.google.com/blog/products/business-intelligence/looker-explore-ai-and-interface-updates/

## 核心主題
Google 更新 Looker Explore 功能，整合 Gemini AI 助手與現代化介面設計，讓企業用戶能以自然語言查詢、自動計算與智慧數據合併等功能，自信地從資料中提取可執行的洞察。

## 關鍵重點
- **AI 助手團隊降低使用門檻**：（1）AI 輔助快速開始（Quick Start）自動生成查詢建議，消除空白畫布的啟動困難；（2）洞察助手（Insight Assistant）透過對話式分析 API，以自然語言修改資料表與視覺化，自動識別欄位、套用篩選與排序；（3）表達式助手（Expression Assistant）將自然語言轉為 Looker Expression 語法，使用者無需學習 Lexp 語法即可建立自訂計算。
- **Merge Query 重新設計的三面板介面**：整合式架構同時顯示「設定聯結」列表、動態欄位選擇器與資料預覽/視覺化三個面板，使用者可在編輯來源查詢時不丢失聯結配置的整體脈絡；智慧聯結建議自動推薦最佳聯結欄位（如州別、月份）、支援貼上既有查詢 URL 快速連結，非 BigQuery 來源的預設列數上限提升至 50,000 列。
- **現代化使用者介面**：可調整大小的欄位選擇器改善長欄位名稱的可讀性；資料表情境選單支援快速切換資料粒度、套用「不為空」篩選與複雜表格計算（如列百分比、累計總和）；即將推出的視覺化樞紐（Visual Pivots）允許拖曳欄位將資料轉列、轉行與聚合。

## 結論
Looker 將對話式 AI 與簡化介面深度融合，讓業務人員能更自信地探索資料、減少對技術團隊的依賴，同時保持資料建模的嚴謹性——這標誌著商業智慧工具從「需要學習的查詢系統」邁向「人人可用的 AI 數據分析夥伴」。
