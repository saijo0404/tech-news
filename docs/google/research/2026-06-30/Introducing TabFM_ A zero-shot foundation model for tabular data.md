# Introducing TabFM: A zero-shot foundation model for tabular data

- **來源**: Google Research Blog
- **發布日期**: 2026-06-30
- **原文連結**: https://research.google/blog/introducing-tabfm-a-zero-shot-foundation-model-for-tabular-data/

## 核心主題
Google 推出 TabFM——首個專為表格數據設計的基礎模型，將表格預測問題重新框架化為**上下文學習**（in-context learning）問題，讓資料科學家在無需特徵工程、超參數調整或模型訓練的情況下，僅透過單次前向傳播即可對全新表格產生高品質預測。

## 關鍵重點
- **打破傳統表格 ML 的瓶頸**：表格數據是企業數據基礎設施的核心，但傳統方法（XGBoost、隨機森林等）在部署時需要資料科學家投入大量時間進行手動超參數最佳化與領域特定的特徵工程。TabFM 將整個表格（包含歷史訓練範例與目標測試行）作為單一統一提示，在推理時直接從上下文中學習欄位間的關係，完全繞過傳統訓練流程。
- **混合注意力架構：交替行/欄注意力 + 行壓縮**：TabFM 結合 TabPFN 與 TabICL 的優勢，透過三大機制處理表格的二维、無序特性：①**交替行與欄注意力**：在多層注意力模組中交替關注欄位（特徵）與行（範例），讓模型原生捕捉複雜的特徵互動與相依性，取代手動特徵編碼。②**行壓縮**：將每行的交叉注意力資訊壓縮為單一稠密向量。③**上下文學習 Transformer**：在壓縮後的行向量序列上進行注意力運算，大幅降低計算成本。
- **全合成數據訓練與 TabArena 卓越表現**：由於真實工業表格數據稀缺且具敏感性，TabFM 完全在數億筆**合成數據**上訓練，使用結構因果模型（SCM）動態生成，涵蓋廣泛的分佈與複雜特徵關係。在 TabArena 活體基準測試中（涵蓋 38 筆分類與 13 筆回歸數據集），TabFM 以即插即用模式單次前向傳播即可持續超越經過大量調校的業界標準監督演算法，而 TabFM-Ensemble（結合交叉特徵、SVD 與 Platt scaling）進一步提升效能。

## 結論
TabFM 將 LLM 領域的 zero-shot 思維帶入表格數據領域，證明了「把表格當上下文」的可行性。它消除了特徵工程、超參數最佳化與重複模型訓練等傳統瓶頸，讓資料科學家的預測工作流程從「手動調校」邁向「即插即用」。Google 更計畫將 TabFM 整合至 BigQuery，未來用戶只需一行 `AI.PREDICT` SQL 指令即可執行進階分類與回歸，大幅降低表格 AI 的採用門檻。
